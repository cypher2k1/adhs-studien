Befehlssätze (3)

Genau genommen müssen alle Summanden, je nach Zustand (an/aus), mit "1" bzw. "0" multipliziert werden:

    

0 * 2^0 + 0 * 2^1 + 0 * 2^2 + 1 * 2^3 + 1 * 2^4 + 0 * 2^5+ 1 * 2^6 ... etc.

    

Berücksichtigt man nur die Zahlenwerte "1" und "0", erhält man die Binärdarstellung dieses Befehls:

    

0001101 ...

    

In dieser Darstellung ist jeder Schalter, an dem eine Spannung anliegt, mit einer 1, alle anderen sind mit einer 0 markiert. Man spricht bei den Schaltern dann von Bits.

    
Da es recht mühsam ist, einen Mikroprozessor auf diese Weise zu steuern, erfand man bald eine Reihe von Abkürzungen für diese Anweisungen: zum Beispiel LD für eine Ladeanweisung, TST für eine Testanweisung etc. Diese Darstellung der Anweisungen nennt man Assembler - und genauso heißen auch die Programme, die aus diesen Anweisungen wieder die entsprechenden Zahlen machen.
