Modifikatoren

Neben abstract gibt es weitere Modifikatoren für Klassen, Attribute und Methoden. Im Wesentlichen handelt es sich dabei um final und static.

    

Mit dem Schlüsselwort final vor der Definition einer Klasse legen Sie fest, dass von dieser Klasse nichts mehr abgeleitet werden darf. Das heißt, es dürfen keine Subklassen mehr gebildet werden. Das macht natürlich nur dann Sinn, wenn die Klasse nicht gleichzeitig als abstract gekennzeichnet wird, denn das würde bedeuten, dass Klassen abgeleitet werden müssen.

  
Definition

Eine Methode, die als final gekennzeichnet wird, darf in abgeleiteten Klassen nicht überschrieben werden.

    

Ein Attribut, das als final gekennzeichnet wird, darf nicht mehr verändert werden, nachdem ihm bei der Deklaration ein Wert zugewiesen wurde. Bei solchen Attributen handelt es sich also um Konstanten.

    

Statische Eigenschaften

Bisher kennen wir Attribute und Methoden immer nur gebunden an einzelne Instanzen: Sowohl jeder einzelne Hund als auch jede einzelne Katze hatte einen eigenen Namen.

    

Mit statischen Eigenschaften können Sie nun Attribute und Methoden erzeugen, die es nur einmal pro Klasse gibt. Sie brauchen also keine Instanz einer Klasse zu erzeugen, um diesem statischen Attribut einen Wert zuzuweisen. Damit übernehmen die statischen Attribute die Rolle von globalen Variablen aus anderen Programmiersprachen. Werden sie zusätzlich noch als final deklariert, kann man sie mit globalen Konstanten vergleichen.

  
Definition
Eine häufig verwendete statische Methode ist die main ()-Methode eines Java-Anwendungsprogramms, die von der virtuellen Maschine aufgerufen wird, bevor irgendein Objekt erzeugt wurde.
