Missbrauch von Interfaces (2)

Das Programm, das wir Ihnen auf der letzten Seite vorgestellt haben, könnte nun so aufgerufen werden:

    
 
System.out.println( berechnen( 5.0, 2.5, ADDITION ));

  

 
Beispiel

Und so sieht es dann aus:

    
  So sieht es aus.
   


Dieses ist aber in gewisser Weise ein Missbrauch des ursprünglichen Konzepts, denn das Programm Rechner implementiert hier die Aktionen im eigentlichen Sinne gar nicht, da das Interface Aktion ja überhaupt keine Methoden definiert, die implementiert werden müssten.

    

Außerdem ist dieses Programm nicht typ-sicher, denn letztlich ist so eine Aktion, wie sie hier definiert ist, ja nur eine Zahl zwischen 1 und 4, auch wenn dieser Zahl ein Name zugeordnet wurde. Ein Programmierer könnte nun aber "aus Versehen" einen Funktionsaufruf wie den folgenden programmieren:

    
 
System.out.println( berechnen( 1, 2, 37));

  

 
Beispiel

Kein Compiler der Welt hätte an diesem Aufruf irgendetwas auszusetzen. Erst beim Ausführen des Programms würde auffallen, dass das Programm überhaupt nicht weiß, was es mit der Zahl 37 anfangen soll: Sollen hier die Werte 1 und 2 addiert werden? Oder multipliziert?
