Enum - Datentypen
Um diese Lücken zu schließen, fügte SUN der Sprache Java in der Version 1.5 die Aufzählungs-Datentypen enum hinzu. Mit diesen neuen Datentypen werden derartige Aufzählungen innerhalb einer Klasse nun wie folgt definiert:
 
Hinweis

 
public class RechnerNeu {

    public enum Aktion { ADDITION,
            SUBTRAKTION, MULTIPLIKATION, DIVISION }

  


 

RechnerNeu.java

Dabei bleibt es dem Benutzer aber verborgen, ob es sich bei den Werten ADDITION, SUBTRAKTION, MULTIPLIKATION sowie DIVISION um Zahlen handelt oder nicht. An die Methode zum Berechnen des Ergebnisses wird auch keine Zahl mehr übergeben, sondern ein Wert vom tatsächlichen Typ Aktion.


    
 
public class Rechner implements Aktion {

    public static double berechnen
            ( double a, double b, Aktion aktion ){
        double result = 0.0;
        if( aktion == ADDITION ){
        result = a + b;
        }

Die eigentlichen Werte des Datentyps Aktion können über die Punktnotation verwendet werden (im Beispiel oben: Aktion.ADDITION), und der neue Funktionsaufruf sieht wie folgt aus:

    
 
System.out.println( berechnen( 5.0, 2.5, Aktion.ADDITION));

Die deutlichsten Unterschiede zu der ersten Version des Programms (Rechner.java) sind:		    
  Der neue Rechner implementiert kein zweckentfremdetes Interface mehr.
  Das Programm ist nun typ-sicher, das heißt, es kann kein Parameter als dritter Wert an die Methode berechnen() übergeben werden, der keine Aktion ist. Der Compiler würde also solch einen Fehler wie im ersten Beispiel sofort entdecken.
