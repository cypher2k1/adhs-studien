PLAF - Definition

Die Abkürzung PLAF steht für Pluggable Look-and-Feel und bedeutet, dass das Aussehen und Verhalten von Swing-Komponenten austauschbar ist. Das ist einer der wesentlichen Unterschiede zwischen AWT und Swing.

  
Definition

Damit kann eine Swing-Applikation z.B. im typischen Windows oder LINUX-Stil gestaltet werden.

    

Folgende Look-and-Feels sind derzeit Teil des JDK: Windows, Motiv und Metal. Die ersten zwei stammen aus den Betriebssystemen Windows und Unix, das dritte ist der Swing-API-Standard.

    

PLAF-Pakete

Die Look-and-Feels werden in Pakete definiert, welche die Klassen enthalten, die für das Zeichnen der verschiedenen Bereiche des User Interfaces zuständig sind.

    

Die Auflistung aller Klassen und Interfaces eines Look-and-Feels finden Sie in dem package

    
javax.swing.plaf
    

Die aktuellen, konkreten Pakete der Look-and-Feel-Standards des JDK sind:

    
  com.sun.java.swing.plaf.windows.WindowsLookAndFeel
  com.sun.java.swing.plaf.motif.MotifLookAndFeel
  javax.swing.plaf.metal.MetalLookAndFeel
