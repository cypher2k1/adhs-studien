CORBA

Der RMI-Mechanismus hat den großen Vorteil, dass außer Java als Programmiersprache und das JDK keine anderen Tools benötigt werden. Genau diese Tatsache wirkt sich mitunter als Nachteil aus. Es ist nämlich nicht ohne weiteres möglich, andere Sprachen einzusetzen. So können in C++ programmierte Server nicht einfach von Clients benutzen werden, die auf Java basieren.

    

Hinzu kommt, dass jeder Client wissen muss, wo, d.h. auf welchem Rechner, die benötigten Server sind. Es ist nicht möglich, einen Server auf einen anderen Rechner zu bewegen, ohne die Clients entsprechend zu ändern. Genauso wenig kann man mehrere Server betreiben, die die gleichen Dienste anbieten, ohne dem Client die Entscheidung aufzubürden, welchen Server er benutzen soll.

    

Eine Alternative zu RMI ist CORBA (Common Object Request Broker Architecture), entwickelt von der OMG (Object Management Group). Auch wenn CORBA älter ist, ist RMI eher eine Vereinfachung von CORBA, als CORBA eine Erweiterung von RMI.
