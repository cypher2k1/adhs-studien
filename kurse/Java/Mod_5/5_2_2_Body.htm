Mehr zu RMI

RMI ist eine der Erweiterungen, die mit JDK 1.1 eingeführt wurden und ermöglicht die Entwicklung verteilter Anwendungen mit Java.

   
Die Client-Server-Verbindung läuft bei RMI über das TCP/IP-Protokoll, das auch fast alle Internetanwendungen, wie WWW, Mail und News verwenden. Die RMIRegistry, die die Aufrufe zwischen Client und Server vermittelt, benutzt normalerweise Port 1099, was manchmal nicht unproblematisch ist, da viele Unternehmen den Zugriff über diese Ports durch eine Firewall unterbinden.
 
Hinweis

Im Folgenden wird ein einfaches Beispiel für einen Server entwickelt, der einen "kleinen" Dienst zur Verfügung stellt, und einen Client, der diesen Dienst nutzt. Versuchen Sie doch einmal, dieses Beispiel Schritt für Schritt selbst zu programmieren.

  
Praxisaufgabe
Zur Vereinfachung gehen wir davon aus, dass Server und Client auf demselben Rechner laufen, was eigentlich dem Sinn von RMI widerspricht. An den entsprechenden Stellen wird auf die notwendigen Änderungen hingewiesen. Auf diese Weise können die Beispiele nachvollzogen werden, ohne dass zwei Rechner parallel zu betreiben sind, die vernetzt sein müssen und über einen eigenen Namen bzw. eine eigene IP-Adresse verfügen.
