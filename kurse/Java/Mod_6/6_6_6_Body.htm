Das Programm wird die SQL-Anweisungen, die an die Datenbank geschickt werden, zur Kontrolle auf den Bildschirm ausgeben. Machen Sie sich einmal klar, was dort passiert. Beachten Sie aber, dass vermutlich andere ID-Werte auf Ihrem Bildschirm stehen, da diese automatisch und zufällig von der Datenbank vergeben werden.
 
Hinweis
 
    

Falls Sie das Beispiel in Access nachvollziehen, schauen Sie doch einmal nach, ob die Daten tatsächlich angekommen sind.

Bitte beachten Sie, dass jeder Datensatz in einer Datenbank nur einmal vorkommen kann. Das heißt, wenn Sie das Programm ein zweites Mal starten, erhalten Sie eine Fehlermeldung, denn die Daten stehen ja bereits in der Datenbank. Sie können die Daten aber zum Beispiel erst mit Access aus der Datenbank löschen und das Programm danach erneut ausführen.
