Die Datei Hibernate-Mapping

Der nächste Schritt besteht darin, ein Mapping zu definieren: Zu jedem Attribut der Klasse Uni wird darin beschrieben, wie die enthaltenen Werte in der Datenbank abgelegt werden sollen.

    
In dem folgenden Mapping wird festgelegt, dass    
  die Objekte der Klasse Uni in der Tabelle UNI in der Datenbank abgelegt werden.
  für jedes Objekt das Attribut id in der Spalte ID der Tabelle abgelegt wird. Beim Anlegen neuer Objekte wird dann automatisch eine aufsteigende Folge von Zahlen als ID vergeben.
  der Name der Uni aus dem Attribut title der Klasse in der Spalte F_NAME der Tabelle abgelegt wird.

   
 
<hibernate-mapping>
    <class name=""hibernatebsp.Uni" table="UNI">
        <id name="id" column="ID">
        <generator class="increment"/>
        </id>
        <property name="title" column="F_NAME" />
    </class>
</hibernate-mapping>

 
 
Uni.hbm.xml


Eine Mapping-Datei wird in Hibernate üblicherweise für jede Klasse definiert, deren Objekte in einer Datenbank abgelegt werden sollen - oder es werden mehrere Mappings in einer einzelnen Datei definiert.

    

Der Vorteil gegenüber der "normalen" Programmierung liegt darin, dass all dies nicht in dem Quelltext des Programms versteckt wird, sondern übersichtlich in einer gesonderten Datei liegt. Dadurch wird es auch möglich, das Mapping der Daten zu ändern, ohne das Programm selber ändern zu müssen.
