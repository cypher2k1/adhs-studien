Signalisieren von Änderungen

Wenn sich andere Objekte als Listener eingetragen haben, muss die JavaBean entscheiden, wann sich der Wert der Eigenschaft ändert und es dann signalisieren. Die entsprechende Methode, die diesen Event auslöst und ihn automatisch an alle eingetragenen Listener versendet, steckt bereits in der Klasse PropertyChangeSupport und heißt firePropertyChange ().

    

Beim Aufrufen der Methode werden zunächst der alte und der neue Wert der Eigenschaft übergeben. Ein weiterer Parameter legt einen Namen für die Eigenschaft fest, damit die eingetragenen Listener wissen, welche sich geändert hat.

    

Das Auslösen oder "Feuern" des Events findet hier in der Methode statt, in der ein neuer Wert der Eigenschaft eingetragen wird: setIntegerValue().Sie merkt sich den alten Wert, der zuletzt in das Feld eingetragen wurde, und überprüft, ob sich dieser von dem neuen Wert des Feldes unterscheidet.

    

In diesem Fall werden die angeschlossenen Listener durch Auslösen eines Events benachrichtigt.

    
 
[...]
if (alt != neu) {
   myPCS.firePropertyChange ("Integer Value",
                              new Integer (alt),
                              new Integer (neu));
}

Beim Auslösen des Events wird ein Name festgelegt, der die geänderte Eigenschaft der JavaBean identifiziert, sowie zwei Objekte, die ihren alten und neuen Wert repräsentieren.
