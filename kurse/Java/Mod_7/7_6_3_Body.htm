Spring Framework

Das Erstellen solcher Java-basierten Webserver-Anwendungen ist jedoch komplizierter als es sich zunächst vielleicht anhört. Um die Komplexität dieser Anwendungen zu vermindern und das Erstellen großer Unternehmensanwendungen in Java zu vereinfachen, sind in letzter Zeit eine Reihe von Konzepten entstanden. Eines davon ist das Spring-Framework.

    
Das Spring-Framework bringt eine Reihe von Komponenten mit, die hier nicht im Detail vorgestellt werden können. Wir möchten Ihnen aber einen kleinen Überblick darüber geben, wie Java bei der Erstellung großer Unternehmensanwendungen eingesetzt wird.    
  Inversion of Control: Hiermit ist gemeint, dass im Grunde nicht mehr Ihr eigener Programmcode die Kontrolle über das hat, was passiert, sondern das Framework wie beispielsweise Spring. Ihr Programm besitzt also keine main()-Methode mehr, sondern Ihr Programm meldet sich beim Framework an, welches dann Ihr Programm aufruft.
  Aspect Oriented Programming: Bei dieser aspektorientierten Programmierung geht es darum, bestimmte Aspekte eines Programms unabhängig von dem Programm selber zu definieren. So können zum Beispiel Attribute von Klassen extern in einer XML-Datei definiert werden und dann von Spring vor dem Kompilieren in die Klassen kopiert werden.
