Klassendiagramme und Argo/UML

Klassendiagramme sind von allen UML-Diagrammen diejenigen, die der Programmiersprache Java am nächsten stehen. Viele Tools bieten eine Generierung von Java-Code aus einem Klassendiagramm an. Außerdem sind Klassendiagramme eine gute Möglichkeit, bestimmte Aspekte eines Java-Codes grafisch darzustellen und zu dokumentieren.

  
Definition

Eine einfache Klasse wird in einem Diagramm durch ein Rechteck symbolisiert, das in drei untereinander liegende Abteilungen aufgeteilt ist. In die oberste Abteilung wird der Name der Klasse zentriert und fett gedruckt eingetragen. In den anderen Abteilungen stehen die Attribute und Operationen. Gibt es keine Attribute und Operationen, fehlen diese Abteilungen manchmal.

    
Die folgenden Beispiele wurden mit dem Programm Argo/UML erstellt, welches einige der Diagrammarten von UML unterstützt. Es ist frei verfügbar, in Java programmiert und kann Java-Code generieren. Eine Installationsversion von Argo/UML finden Sie auf dieser CD-ROM im Ordner Software.
