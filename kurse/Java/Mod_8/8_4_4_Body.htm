Definition der Klassen und Quellcode-Generierung

Die Klassennamen sollen laut einer UML-Konvention Substantive in Einzahl sein und mit einem Großbuchstaben beginnen - Attribute und Operationen beginnen mit einem Kleinbuchstaben.

    

Die Namen von abstrakten Klassen werden kursiv gedruckt. Die Vererbung wird durch einen Pfeil von der Unterklasse zur Oberklasse symbolisiert. Dabei sind die Pfeilspitzen ungefüllte Dreiecke.

    

Abbildung: Klassendiagramm - Beispiel 2
    
In den zwei darunter liegenden Abteilungen werden die Attribute und Methoden eingetragen. Tools können jedoch auch weitere, herstellerabhängige Abteilungen definieren.
 
Hinweis

Das Beispiel wird nun um die abstrakte Klasse Person ergänzt.

    
 
public abstract class Person {}
